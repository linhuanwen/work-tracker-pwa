## 1. 准备工作

### 你需要的

| 条件 | 说明 |
|------|------|
| **WPS Office**（公司电脑） | 登录你的 WPS 账号，开通云文档功能 |
| **WPS App**（安卓手机） | 同一账号 |
| **Chrome / Edge 浏览器** | 公司电脑目前使用的浏览器即可，需较新版本（支持 PWA） |
| **个人电脑**（可选） | 仅用于运行 AI 润色脚本，不需要的话可跳过 |
| **Python 3.11+**（可选） | 同个人电脑，仅 AI 润色需要 |

### 建好同步文件夹

1. 打开公司电脑的 WPS Office
2. 确保已登录 WPS 账号，左侧能看到"云文档"
3. 在 `WPS云文档` 文件夹下新建一个子文件夹，取名如 **`工作清单`**
4. 这个文件夹会自动同步到你的手机（WPS App）和个人电脑（如果也装了 WPS 并登录同一账号）

> 💡 **为什么用 WPS 云文档文件夹？** 因为这样电脑上 PWA 读写一个普通 JSON 文件，WPS 自动帮你同步到所有设备。不需要自己搭服务器。

## 2. 安装 PWA

### 2.1 启动开发服务器（仅第一次）

项目已开发完成，你需要在公司电脑上跑一个本地开发服务器：

```bash
# 在项目根目录打开终端
cd "D:\Vibe coding\06 rili"
npm run dev
```

你会看到类似这样的输出：

```
  VITE v6.4.3  ready in 300 ms

  ➜  Local:   http://localhost:5173/
```

### 2.2 打开 PWA

浏览器访问 `http://localhost:5173/`

首次打开的画面：

- 中间一个大按钮"**选择工作文件夹**"
- 点击它 → 会弹出文件夹选择对话框
- 导航到你的 WPS 云文档文件夹（在 Windows 资源管理器路径类似 `C:\Users\你的用户名\Documents\WPS云文档\工作清单`）
- 选择后，PWA 在这个文件夹里创建 `data.json`，一切数据都存在这里面

### 2.3 安装为独立窗口（重要！）

这是解决"浏览器标签容易手滑关掉"的关键步骤：

**Chrome：**
1. 点击地址栏右边的 **⊕ 安装图标**（或者省略号菜单 → "安装工作清单..."）
2. 弹出的对话框点"安装"
3. 桌面右下角任务栏会出现一个独立的"**工作清单**"窗口图标
4. 右键图标 → **钉到任务栏**，以后直接点击打开

**Edge：**
1. 省略号菜单 → "应用" → "**将此站点作为应用安装**"

**安卓手机：**
1. 浏览器打开 PWA 地址
2. 菜单 → "**添加到主屏幕**"
3. 桌面会出现"工作清单"图标，点击打开，像普通 App 一样

> 安装后，这个窗口没有地址栏、没有标签栏、没有浏览器按钮。监控软件看到的只是"浏览器在正常使用"，但你的体验和原生 App 相同。

### 2.4 如果觉得每次要终端启动太麻烦

可以写一个 `.bat` 文件放在桌面，双击启动：

```bat
@echo off
cd /d "D:\Vibe coding\06 rili"
start "" "http://localhost:5173"
npm run dev
```

> ⚠️ 关闭终端窗口 = 停止服务器 = PWA 打不开。只要不关终端就行，最小化即可。

## 3. 每日使用

### 3.1 界面概览

打开后的主界面分三区：

```
┌────────────────────────────────────┐
│  工作清单     ← 标题栏               │
│────────────────────────────────────│
│  🔴 紧急处理 (最多5条)               │  ← 紧急区
│  ┌─────────────────────────────┐   │
│  │ ☐ XX部门资格审查 [今日截止]    │   │  ← 红色左边框
│  │   👤领导交办  📊审查15人次     │   │
│  ├─────────────────────────────┤   │
│  │ ☐ 晋升材料提交确认            │   │
│  └─────────────────────────────┘   │
│                                    │
│  🟡 重要                            │  ← 重要任务区
│  ☐ 年度晋升专项支持                 │
│    ████████░░ 67% (8/12 子任务)    │
│                                    │
│  🟢 日常                            │  ← 日常任务区
│  ☑ 周报提交                        │
│  ☑ 会议纪要整理                     │
│────────────────────────────────────│
│        ＋                          │  ← FAB 新建按钮（右下角）
│────────────────────────────────────│
│  📋列表  📁项目  📊小结  ⚙设置      │  ← 底部导航
└────────────────────────────────────┘
```

### 3.2 添加任务

1. 点击右下角蓝色 **＋** 按钮
2. 填写表单：

| 字段 | 说明 | 必填 |
|------|------|------|
| 标题 | 任务名称，如"XX部门资格审查" | ✅ |
| 分类 | 下拉选择：人员调配 / 内部招聘 / 奖惩管理 / 绩效管理 / 劳动关系 / 领导交办 / 其他 | ✅ |
| 优先级 | 🔴紧急 / 🟡重要 / 🟢日常 | ✅ |
| 截止日期 | 过期未完成的任务会自动提醒 | 选填 |
| 量化产出 | 可添加多条，每条包含 指标名 + 数值 + 单位（如"审查 15 人次"） | 选填 |

3. 点击"添加"按钮

> 💡 **键盘快捷键**：在输入框中按 `Enter` 快速添加

### 3.3 查看和编辑任务

- **展开详情**：点击任务卡片任意位置 → 展开显示完整信息（量化产出、子任务、备注、时间戳）
- **快速勾选完成**：点击任务左边的复选框 → 标记完成（带删除线 + 淡化效果）
- **更多操作**：展开后可以进行：
  - 切换状态（待办 → 进行中 → 已完成 → 已取消）
  - 修改优先级别
  - 添加/编辑量化产出
  - 添加子任务
  - 修改标题、分类、截止日期、备注
  - 打开"领导交办"开关（出现来源、交办时间、时限字段）
  - 设置跨年休眠

### 3.4 紧急区规则

- 优先级 = **紧急** 的任务自动进入顶部紧急区
- 紧急区最多展示 **5 条**——第 6 条添加时自动降级为"重要"，并弹提示
- 紧急任务在卡片上有**红色左边框** + 🔴 图标
- 一旦标记完成或取消，自动离开紧急区

### 3.5 领导交办任务

打开编辑面板 → 点击"**领导交办**"开关 → 展开三个字段：

- **交办来源**：如"张处长"或"分管领导"
- **交办时间**：领导交代的日期
- **完成时限**：领导要求的截止日期

标记后的任务卡片上出现红色 **"交办"** 徽章，一目了然。

### 3.6 量化产出录入

在编辑面板的"量化产出"区域：
1. 点击"＋ 添加产出"
2. 填写指标名（如"审查人次"）、数值（15）、单位（人次）
3. 可以添加多条
4. 卡片上以灰色小标签展示，如 `审查 15 人次` `通过 14 人次`

> **为什么重要？** 这些数字是年终报告的基础——年底你的"全年助力晋升 12 人""审查 138 人次"就是从这里一个个加起来的。

### 3.7 子任务 & 项目（进阶）

如果你有一个长期任务，拆分子任务来追踪：

**在任务编辑面板底部：**
1. 点击"添加子任务"
2. 输入小步骤名称（如"收集材料"→"逐一审核"→"整理报告"）
3. 每天完成一个小步骤就勾一下
4. 卡片上自动出现进度条：`████░░░░ 3/8`，一眼知道推进了多少
5. 全部子任务勾选后，不会自动完成——你手动点"**确认完成**"按钮

**创建项目：**
1. 底部导航 → "📁 项目"
2. 新建项目（标题 + 分类 + 起止日期）
3. 在任务编辑面板中，"关联项目"下拉选择对应的项目
4. 项目视图里可以看到所有关联任务和整体进度

> 💡 项目页面的进度条颜色会变：红色（0-33%）→ 黄色（34-66%）→ 蓝色（67-99%）→ 绿色（100%完）

## 4. 每周小结

### 4.1 触发生成

- 底部导航 → "📊 小结" → 顶部切换到"周小结"
- 点击"**生成本周小结**"按钮
- 或者：到了你设定的周结日（默认周五），系统在设置页显示提示

### 4.2 周小结的四段式

| 段落 | 内容 | 是否可以编辑 |
|------|------|-------------|
| **一、本周完成任务** | 按分类分组列出本周所有已完成任务 + 量化产出 | ✅ 可编辑 |
| **二、长期项目推进** | 本周有进展的项目 + 进度变化（如 58% → 67%） | ✅ 可编辑 |
| **三、下周计划** | 自动拉入未完成任务 + 你手动补充的条目 | ✅ 可编辑 |
| **四、需协调事项** | 标记了"阻塞"的进行中任务 | ✅ 可编辑 |

### 4.3 "下周计划"特殊机制

在第三段手动添加的条目（点"+ 添加计划"后输入内容），**会自动在任务清单中创建对应的新任务**。不需要输入两遍。

比如你在周小结里写了：
> 下周计划 → "+ 添加计划" → "完成年度人才盘点报告初稿"

系统自动在任务列表里创建一条标题为"完成年度人才盘点报告初稿"的待办任务。

### 4.4 复制分享

标题栏右侧有"**📋 复制**"按钮，一键把整个周小结复制为纯文本，粘贴到 OA、邮件、微信。

### 4.5 AI 润色（可选）

点击"**✨ 请求 AI 润色**"按钮 —— 这只是在 data.json 里标记一个标志位。实际的 AI 润色在个人电脑上运行（见 [第 7 节](#7-ai-润色脚本)）。

## 6. 年度报告

### 触发

底部导航 → "小结" → 切换到"年度报告" → 选择年份 → "生成年度报告"

### 六大维度归纳

年度报告不按时间罗列，而是按你作为人力资源经办人的工作板块归类：

1. **人员配置** — 调配方案、编制调整、岗位分配等
2. **内部招聘 / 晋升晋等** — 资格审查、材料辅导、答辩模拟、助力晋升
3. **奖惩管理** — 奖励申报、处分执行等
4. **绩效管理** — 考核组织、数据汇总、结果反馈等
5. **劳动关系** — 合同续签、入离职办理等
6. **领导交办** — 上级交代的各类事项

每个维度自动展示：

- **工作量统计**：全年该分类共完成多少任务
- **量化产出合计**：所有量化指标的全年加总（如"全年审查 138 人次，助力晋升 12 人"）
- **关键任务列表**：该分类下所有已完成任务

每个维度底部有"**关键业绩提炼**"文本区，你手动写一两句话总结这个维度的核心成果。

### 附表（自动计算）

报告末尾附两张表：

1. **月度趋势表** — 12 行 × 7 列（1-12月 × 各分类），每格是当月该分类完成的任务数
2. **全年量化产出总表** — 按指标名汇总全年所有量化产出

### 年度报告的使用场景

- **给领导看**：按维度展示工作量 + 关键业绩 → 直接复用为述职报告
- **给考评用**：量化产出数据来自日常记录 → 真实可查
- **给自己看**：反思空间 + 附表 → 作为下一年规划的参考
- **跟上级计划"对纲"**：复制年度报告内容 + 上级年度人力工作计划 → 一起发给 AI（ChatGPT/Claude 等），让它帮你按纲要重新组织，体现"组织导向下的个人贡献"

## 7. AI 润色脚本

### 什么时候用

只有当你在 PWA 里点了"请求 AI 润色"按钮后才需要用。这是一个**完全独立**的 Python 命令行工具，跑在你的个人电脑上。

### 前提条件

- 个人电脑安装了 **Python 3.11+** 和 WPS Office（已登录同一账号）
- 有一个 AI API 账号（DeepSeek 或通义千问），获取 API Key

### 安装

```bash
# 进入 scripts 目录
cd scripts/

# 安装依赖（只需要一个库）
pip install requests
```

### 配置

在 `scripts/` 目录下创建 `.env` 文件（参考已有的 `.env.example`）：

**DeepSeek：**
```env
AI_API_KEY=sk-你的密钥
AI_ENDPOINT=https://api.deepseek.com
AI_MODEL=deepseek-chat
```

**通义千问：**
```env
AI_API_KEY=sk-你的密钥
AI_ENDPOINT=https://dashscope.aliyuncs.com/compatible-mode/v1
AI_MODEL=qwen-turbo
```

### 运行

```bash
cd scripts/
python polish.py "D:\Documents\WPS云文档\工作清单\data.json"
```

脚本会列出所有待润色的条目，你确认后逐项处理。润色结果直接写回 JSON，WPS 自动同步到公司电脑。

### 润色效果

AI 只润色**语言表述**，不增删事实：

| 润色前 | 润色后 |
|--------|--------|
| 这周搞定了XX部门的资格审查，审了15个人，过了14个 | 本周完成XX部门资格审查：审查 15 人次，通过 14 人次 |
| 推进了一下晋升专项，完成了材料辅导和答辩模拟 | 持续推进年度晋升支持专项，本周完成材料辅导及答辩模拟各 1 次 |

## 9. 数据同步

### 工作原理

```
公司电脑修改 data.json → WPS自动上传云端 → 手机WPS自动下载 → 手机PWA看到最新
```

你不需要手动做任何同步操作。WPS 云文档在后台自动完成。

### 跨年休眠任务同步

休眠任务（`isCrossYear=true` + `hibernateUntil` 设了未来日期）在各个设备上都会被过滤掉，只有 `hibernateUntil` 时间到了才自动出现。

## 10. 常见问题

### Q: 打开 PWA 时显示"无法连接到 localhost:5173"

A: 开发服务器没有启动。重新运行 `npm run dev`。

### Q: 点击"选择工作文件夹"没反应

A: 确认浏览器支持 File System Access API（Chrome 86+ 或 Edge 86+ 都支持）。如果浏览器版本过旧，升级浏览器。

### Q: 手机打开 PWA 后数据是空的

A: 确保：
1. 手机 WPS App 已登录同一账号
2. 手机 WPS 云文档中能看到 `工作清单/data.json` 文件
3. 手机 PWA 需要像电脑一样"选择工作文件夹"——导航到 WPS 下载目录中的 data.json 所在文件夹

### Q: 数据安全吗？会不会被别人看到？

A: 
- 数据存在你自己的 WPS 云文档里，不经过任何第三方服务器
- AI 润色仅在你个人电脑上运行时才调用 AI API，且只发送需要润色的文字段落
- 敏感信息（如某些人名）可以在编辑周小结时删除，再点 AI 润色

### Q: 手机端 PWA 能不能离线使用？

A: 可以。PWA 已配置离线缓存策略——静态资源（界面、样式、脚本）在首次加载后缓存。但离线时编辑的数据需要在联网后才能同步到 WPS 云端。

### Q: 月底了上个月的数据还能改吗？

A: 可以。周小结和月小结生成后随时可重新编辑——你之前确认过要"事后补录"的灵活性。

### Q: 怎么用年度报告做述职？

A: 流程是：
1. 在 PWA 里生成年度报告
2. 复制六个维度的内容
3. 把你单位的"上级年度人力工作计划"一起发给 AI（ChatGPT/Claude/通义千问等）
4. 告诉 AI："请按这份年度计划为纲，把以下工作成果重新组织成述职报告，体现个人贡献与组织发展的一致性"
5. AI 输出后你再修改润色

### Q: 项目完全离线使用不用 WPS 可以吗？

A: 可以。选择文件夹时选任意本地文件夹（如桌面或文档文件夹），数据就只存在那个文件夹里。但这样就只能在电脑上用，手机同步不了。
